# OpenAPI / REST Contract: JSON Column Surface

**Entity**: `Profile` (table `dbo.profiles`)
**REST base path** (test fixture): `/api/Profile`
**Scope**: How a SQL Server `JSON` column appears in the OpenAPI
schema generated by DAB, and the exact request/response shapes for
every operation defined by the spec's user stories.

This document defines the **client-observable contract**. Internal
serializer behavior may evolve as long as these shapes hold.

---

## Schema component (OpenAPI 3)

DAB emits the JSON column as a plain string property â€” no `format`,
no `pattern`, no `x-dab-` extension (FR-001, FR-013).

```yaml
components:
  schemas:
    Profile:
      type: object
      properties:
        id:
          type: integer
          format: int32
        metadata:
          type: string
          nullable: true
      # 'id' is omitted from 'required' for create payloads (IDENTITY).
```

## Operations

For brevity, requests and responses below show only the JSON-relevant
fields. Standard DAB response envelopes (e.g., `value[]` for collection
GETs) are unchanged.

### GET /api/Profile/id/1

**Response 200**

```json
{
  "value": [
    {
      "id": 1,
      "metadata": "{\"role\":\"admin\",\"tier\":3}"
    }
  ]
}
```

`metadata` is a JSON **string** whose characters are the JSON text
returned by `SELECT metadata FROM profiles WHERE id = 1`. DAB does not
parse it into a nested object (FR-002, User Story 2).

### POST /api/Profile

**Request body**

```json
{
  "metadata": "{\"role\":\"guest\"}"
}
```

**Response 201**

```json
{
  "value": [
    { "id": 6, "metadata": "{\"role\":\"guest\"}" }
  ]
}
```

**Rejected â€” nested object on input** (FR-004; spec Clarification Q1)

```json
{ "metadata": { "role": "guest" } }
```

â†’ Response `400 Bad Request` with body identifying that `metadata`
must be a JSON-encoded string.

**Rejected â€” malformed JSON** (FR-007; spec Clarification Q2)

```json
{ "metadata": "{not valid json" }
```

â†’ Response `400 Bad Request`. Body identifies that `metadata` is not
valid JSON. The underlying SQL Server error number (in the 13600 range)
is NOT leaked in production mode.

### PUT /api/Profile/id/1

**Request body**

```json
{ "metadata": "{\"role\":\"owner\"}" }
```

**Response 200** â€” full row with new `metadata` (User Story 5).

### PATCH /api/Profile/id/1

**Request body**

```json
{ "metadata": "{\"role\":\"owner\"}" }
```

**Response 200** â€” same as PUT for the touched field; other columns
unchanged (User Story 5).

**Clear to NULL** (User Story 8)

```json
{ "metadata": null }
```

â†’ `metadata` becomes SQL `NULL`; response renders `"metadata": null`.

### DELETE /api/Profile/id/1

**Response 204** (User Story 6). No special handling for JSON.

## Filter & orderby contract (User Story 9, FR-009/FR-010)

### Allowed `$filter` operators on JSON columns

| Operator | Status | Behavior |
|----------|--------|----------|
| `eq` | Allowed | Compares stored JSON text as string. |
| `ne` | Allowed | Compares stored JSON text as string. |
| `eq null` | Allowed | Translates to `IS NULL`. |
| `ne null` | Allowed | Translates to `IS NOT NULL`. |
| `gt`, `lt`, `ge`, `le` | **Rejected** | 400; "operator not supported on JSON column `metadata`". |
| `contains`, `startswith`, `endswith` | **Rejected** | 400; same message. |
| JSON-path predicates (`JSON_VALUE(...)`, `$.foo`, etc.) | **Rejected** | Either rejected by OData parser before reaching DAB, or by the JSON-column gate. |

**Example â€” allowed**

```
GET /api/Profile?$filter=metadata eq '{"role":"admin","tier":3}'
GET /api/Profile?$filter=metadata eq null
```

**Example â€” rejected**

```
GET /api/Profile?$filter=contains(metadata,'admin')
```

â†’ `400 Bad Request`: `{"error":{"code":"BadRequest","message":"Operator 'contains' is not supported on JSON column 'metadata'."}}`

### `$orderby` on JSON columns

Allowed. Orders by the underlying string representation of the stored
JSON text. Documented as "string-order, not semantic-JSON order"
(User Story 9, FR-010).

```
GET /api/Profile?$orderby=metadata
GET /api/Profile?$orderby=metadata desc
```

## Error envelope summary

| Cause | HTTP status | DAB sub-status |
|-------|-------------|----------------|
| `metadata` sent as a nested object/array (FR-004) | 400 | BadRequest |
| Malformed JSON text in `metadata` (FR-007) | 400 | DatabaseInputError-or-BadRequest |
| Unsupported `$filter` operator on JSON column (FR-009) | 400 | BadRequest |
| Target SQL Server build lacks JSON type (FR-016) | 500 (startup) or upstream SQL error at request time | (unmapped â€” native error surfaces) |
| Permission denied | 403 | unchanged |
| Row not found | 404 | unchanged |
| Delete on row with JSON | 204 | unchanged |
